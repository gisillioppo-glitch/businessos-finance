# Dashboard Pilot Plan Page v0.1

## Product meaning

This block brings the private pilot plan into the private dashboard. It gives operators and executive owners a visual, read-only view of the 14-day pilot plan, roles, workflow, daily rhythm, success criteria, exit decisions, and protected boundaries.

## Why it matters

A pilot plan should not live only in Markdown. The dashboard page makes the plan visible before and during a private pilot, keeping the team aligned around owner, scope, timeline, evidence, and safe operating limits.

## Current capabilities

- Adds `Pilot Plan` to private dashboard navigation.
- Reads latest `reports/private_pilot_plan_YYYY-MM-DD.md`.
- Shows plan status, pilot length, owner, workflow, and phase count.
- Lists 14-day timeline, daily rhythm, roles, success criteria, exit decisions, and boundaries.
- Keeps the page read-only; the artifact is generated by `python cli.py private-pilot-plan`.

## Safety boundary

The page does not enable deployment, email delivery, or workflow execution. It only displays the approved pilot planning artifact.

## Next step

Use this page during pilot kickoff and review. Later, a pilot status tracker can add live progress against each phase.
